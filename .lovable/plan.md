

## Fix `cn_payment_pkey` Duplicate Key Error

### Root Cause
Two problems:

1. **Sequence out of sync**: The identity sequence `cn_payment_payment_sequence_no_seq` has `last_value=2`, and existing rows already have values 1 and 2. The next insert will try value 3 which should work — BUT if the sequence was previously manually overridden or the identity column is `BY DEFAULT` (not `ALWAYS`), manually provided values can still conflict.

2. **`usePaymentEntry.ts` manually sets `payment_sequence_no`**: The `addDetailRow` function (line 203-208) calculates `nextSeq` manually and passes it in the insert. Since the column is `GENERATED BY DEFAULT AS IDENTITY`, providing a value overrides the sequence — but doesn't advance it. This causes the sequence to fall behind, and future inserts (with or without manual values) collide.

3. **`PaymentHistoryManagement.tsx`** (line 148-151) also manually sets `payment_sequence_no` when splitting payments.

### Plan

**1. Database migration — reset the sequence**
- Run: `SELECT setval('cn_payment_payment_sequence_no_seq', (SELECT COALESCE(MAX(payment_sequence_no), 0) FROM public.cn_payment));`
- This ensures the sequence is in sync with existing data.

**2. Fix `usePaymentEntry.ts` — stop manually setting `payment_sequence_no`**
- In `addDetailRow`, remove `payment_sequence_no: nextSeq` from the insert object.
- Let the database identity column auto-generate it.
- After insert, the returned `data` will contain the generated value.

**3. Fix `PaymentHistoryManagement.tsx` — stop manually setting `payment_sequence_no`**
- In the split payment logic (line 148-151), remove `payment_sequence_no: maxSeq + i` from new row inserts.

**4. Verify receipt generation works end-to-end**
- The `PaymentDataEntry.tsx` insert at line 174 already omits `payment_sequence_no` — this is correct.
- After the sequence reset and code fixes, the full flow (header → details → receipt → print log) should complete without errors.

### Files to Change
- **Migration SQL**: Reset `cn_payment_payment_sequence_no_seq` to current max
- **`src/hooks/usePaymentEntry.ts`**: Remove manual `payment_sequence_no` from `addDetailRow`
- **`src/pages/cashier/PaymentHistoryManagement.tsx`**: Remove manual `payment_sequence_no` from split inserts
